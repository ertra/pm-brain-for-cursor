# Company context

This folder holds **your filled-in** company context. Skills read from here when drafting briefs, PRDs, and analyses so claims stay grounded in shared facts.

> This folder ships **empty on purpose** — only this `README.md` is here. The context files below are generated by `/start`.

---

## How to populate this folder

**Automated (recommended):** run **`/start`** in Cursor with your company name and website URL. The [`start`](../.cursor/skills/start/SKILL.md) skill researches the public web via the [`company-context-researcher`](../.cursor/agents/company-context-researcher.md) subagent and writes the files for you.

**Manual:** copy the blank templates from [`02-templates/context/`](../02-templates/context/) into this folder and replace all placeholders.

Re-running `/start` **overwrites** the files in place (no backup) — commit your edits to version control before re-running.

---

## What `/start` creates

After `/start` completes, `01-context/` will look like this:

```
01-context/
├── README.md               # this file (not modified by /start)
├── 01-about-company.md     # ICP, buyer vs user, GTM, compliance, brand voice, pains
├── 02-products.md          # Surfaces, platform capabilities, integrations, modules, tiers
├── 03-product-vision.md    # North star, strategic themes, product principles
├── 04-competition.md       # Competitors, win/loss, status quo alternatives, pricing
└── 05-glossary.md          # Canonical terms (only written if research surfaces glossary content)
```

| File | Purpose |
|------|---------|
| `01-about-company.md` | ICP, buyer vs user, GTM, compliance, brand voice, pains table |
| `02-products.md` | Surfaces, platform capabilities, integrations, modules, tiers |
| `03-product-vision.md` | North star, strategic themes, product principles |
| `04-competition.md` | Competitors, win/loss, status quo alternatives, pricing bands |
| `05-glossary.md` | Canonical terms (optional — only created when research returns glossary content) |

Every generated file includes YAML frontmatter (`title`, `document`, `last_updated`, `confidence`, `last_verified`, `source_notes`) so agents can cite sources and flag stale context. Cells the research could not confirm are marked **`To be verified`** — review and correct before sharing externally.
